# TICKET-042: 【Mobile】シフト希望送信画面

## 担当
Frontend

## 概要
バイトがシフト希望を簡単に送信できる画面を作成する。
**第2優先機能** - 「極限まで楽にする」がコンセプト。

## リポジトリ
`ikki-dali/airport-shift-mobile`

## 画面要件

### リクエストタブ（app/(tabs)/request.tsx）

#### コンセプト
- **とにかく簡単に** - 最小限のタップで希望を送信
- **時間軸で自由に選べる** - A・Bグループ等の枠ではなく、時間で選ぶ

#### ヘッダー
- 対象年月（例: 2024年2月の希望）
- 月切り替え

#### メインエリア：カレンダー形式
- 月間カレンダーを表示
- 各日をタップで希望入力モーダルを開く
- 希望済みの日にはマーク（✓やドット）

#### 希望入力モーダル
タップした日付に対して：
1. **希望区分**を選択
   - ○ 出勤可能
   - △ できれば休み
   - × 出勤不可
2. **時間帯**を選択（出勤可能の場合）
   - 終日OK
   - 午前のみ（〜13:00）
   - 午後のみ（13:00〜）
   - カスタム（開始時間〜終了時間）
3. **備考**（任意）
   - フリーテキスト

#### 送信機能
- 「この月の希望を送信」ボタン
- 送信前に確認ダイアログ
- 送信成功時にトースト表示

### データ保存
```typescript
// shift_requestsテーブルに保存
const { error } = await supabase
  .from('shift_requests')
  .upsert({
    staff_id: staffId,
    year_month: '2024-02',
    date: '2024-02-05',
    preference: '○', // ○, △, ×
    time_slot: 'all_day', // all_day, morning, afternoon, custom
    start_time: null, // customの場合のみ
    end_time: null,   // customの場合のみ
    notes: '備考テキスト'
  })
```

- カレンダーは大きめ（タップしやすく）
- 希望入力は1〜2タップで完了
- 視覚的にわかりやすい（○△×を色分け）
