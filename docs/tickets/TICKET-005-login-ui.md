# TICKET-005: ログインUI実装

## 担当
Frontend

## 背景
- TICKET-004で認証基盤が構築される
- ログインページのUIが必要
- Email/Password方式

## 要件

### 1. ログインページ (`/login`)
- メールアドレス + パスワードフォーム
- ログインボタン
- エラー表示（認証失敗時）
- ログイン成功時はホームページ（`/`）にリダイレクト

### 2. ログアウトボタン
- 既存のヘッダー/ナビゲーションにログアウトボタン追加
- ログアウト後は`/login`にリダイレクト

### 3. デザイン
- 既存UIと統一（shadcn/ui使用）
- シンプルなフォーム、レスポンシブ対応
- ロゴ/タイトル表示

## 依存
- TICKET-004（認証基盤）が先に完了していること

- [ ] /loginページが存在し、フォームが動作する
- [ ] ログイン成功でホームにリダイレクト
- [ ] ログイン失敗でエラー表示
- [ ] ログアウトボタンが動作する
- [ ] `npm run build`が成功
