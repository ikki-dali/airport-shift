# TICKET-037: ダッシュボード大幅リデザイン

## 担当
Frontend

## 概要
マネージャーが「見るだけで状況がわかる」ダッシュボードに改修する。

## 現状の問題
- 「ダッシュボード」「シフトの概要」など不要な見出しがある
- 情報が散らばっていて一目で把握しにくい
- Today/This Week/This Monthの切り替えがない

## 新しいデザイン要件

### 1. ヘッダー部分（削除）
- 「ダッシュボード」「シフトの概要」などの見出しテキストは**削除**

### 2. 上部統計バー（小さく表示）
コンパクトな横並びで以下を表示：
- 充足率（%）
- 人数不足の日数
- 承認待ち件数（バッジ）

### 3. メインエリア（タブ切り替え）

#### タブ構成
```
[Today] [This Week] [This Month]
```

#### Today タブ
- 今日の出勤者一覧
- スタッフ名、勤務地、時間帯を表示
- 人が足りていない枠があれば赤ハイライト

#### This Week タブ
- 今週のシフト一覧（月〜日）
- 日ごとの出勤者数を表示
- 空いてる日（人手不足）は**赤ハイライト**

#### This Month タブ
- **カレンダー表示**
- 各日に出勤者数を表示
- 空いてる日（人手不足）は**赤ハイライト**
- クリックでその日の詳細を見れると良い（オプション）

### 4. 削除する要素
- クイックアクションカード（シフト作成、スタッフ管理などのボタン群）
- システム情報カード（登録スタッフ、配属箇所、勤務記号）
- 今週のシフトカード（タブに統合）

## 完了条件
- [ ] 見出しテキストが削除されている
- [ ] 上部に統計バーが小さく表示されている
- [ ] Today/This Week/This Monthのタブ切り替えが動作する
- [ ] This Monthがカレンダー表示になっている
- [ ] 人手不足の日が赤ハイライトされている
- [ ] ビルドが通る
- [ ] レスポンシブ対応（モバイルでも見やすい）

## 参考
- 既存の `app/page.tsx` を大幅改修
- shadcn/ui の Tabs コンポーネントを使用
- カレンダーは shadcn/ui Calendar または自作

## 技術メモ
- 既存のデータ取得ロジック（getShiftsWithDetails等）は流用可能
- locationRequirements との比較で人手不足を判定
