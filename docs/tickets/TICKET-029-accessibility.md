# TICKET-029: アクセシビリティ改善

## 担当
Frontend

## 背景
- 業務アプリとしてキーボード操作とスクリーンリーダー対応が必要
- 主要な改善ポイントを対応

## 要件

### 1. aria-label追加
- アイコンのみのボタンにaria-label追加（編集、削除、通知ベル等）
- フォームのlabel要素とinputの紐付け確認（htmlFor/id）

### 2. キーボードナビゲーション
- モーダル/ダイアログにフォーカストラップ確認（shadcn/uiのDialogは対応済みのはず）
- Escキーでモーダル/ダイアログが閉じる確認
- Tabキーで主要要素を移動できる確認

### 3. コントラスト・視認性
- テキストのコントラスト比確認（4.5:1以上）
- フォーカスリングが見える確認（outline-ringスタイル）
- ステータスバッジの色だけでなくテキストでも意味が伝わる確認

### 4. セマンティックHTML
- main/nav/header/footer等のランドマーク要素確認
- 見出しレベル（h1→h2→h3）の順序確認
- テーブルにcaption/theadが適切に使用されている確認

## 注意事項
- shadcn/uiは基本的にアクセシブルだが、カスタムコンポーネントの確認が必要
- 完璧を求めず、主要な問題を解決する
- axe-coreやLighthouse等でのチェック推奨
