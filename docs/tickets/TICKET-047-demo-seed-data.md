# TICKET-047: デモ用シードデータ作成

## 担当
Frontend

## 概要
クライアントへのデモ用に、リアルなダミーデータを作成する。
150人のスタッフ、シフトデータを生成。

## リポジトリ
`ikki-dali/airport-shift`

## データ要件

### スタッフ（150人）
| 種別 | 人数 | 特徴 |
|------|------|------|
| 契約社員 | 30人 | 月170h程度、週5勤務 |
| パート | 120人 | 月120h以下、週2-4勤務 |

- 日本人名（田中、山田、佐藤など）
- 社員番号（連番）
- メールアドレス（ダミー）
- 勤務地の割り当て

### シフトデータ
- 今月と来月分
- **1日あたり43人**がシフトに入る
- 日によって若干の過不足あり（デモ用）
  - 充足している日
  - 人手不足の日（赤ハイライト確認用）
  - 承認待ちのシフト（バッジ確認用）

### 勤務地
- 既存の勤務地を使用
- なければ作成（第1ターミナル、第2ターミナル等）

## 実装方法

### シードスクリプト
`lib/seed/demo-data.ts` を作成

```typescript
export async function seedDemoData() {
  // 1. スタッフ150人を作成
  // 2. 勤務地を確認/作成
  // 3. シフトデータを生成（1日43人ベース）
  // 4. 一部を承認待ち状態にする
}
```

### 実行方法
`/admin/seed` ページにボタン追加
- 「デモデータを生成」ボタン
- 確認ダイアログ付き
- 既存データは削除してから生成

## ダッシュボード改修
- Todayセクションに「43人中 ○人確定」を表示
- 1日の必要人数を43人として計算
