# TICKET-049: 【Mobile】A〜G時間帯グループ選択

## 担当
Frontend

## 概要
シフト希望送信時に、A〜Gの時間帯グループを選択できるようにする。
「フルで可能」な人は全グループ選択、特定時間帯だけの人は該当グループのみ選択。

## リポジトリ
`ikki-dali/airport-shift-mobile`

## 時間帯グループ定義

| グループ | 時間帯 | 概要 |
|---------|--------|------|
| **A** | 4:00〜13:00頃 | 早朝帯 |
| **B** | 6:00〜15:30頃 | 朝帯 |
| **C** | 6:45〜22:00頃 | 日中帯 |
| **D** | 12:00〜23:00頃 | 午後帯 |
| **E** | 14:15〜23:00頃 | 夕方帯 |
| **F** | 16:30〜23:00頃 | 夜帯 |
| **G** | 19:00〜翌朝 | 深夜帯 |

## UI要件

### リクエスト画面に追加

```
┌─────────────────────────────────────┐
│ ① 基本設定                          │
│    [○ 出勤可能] [△] [×]            │  ← 既存
├─────────────────────────────────────┤
│ ② 希望時間帯（○の場合）【NEW】      │
│                                     │
│    [✓ 全時間帯OK]  ← デフォルト     │
│                                     │
│    または特定時間帯を選択:           │
│    [A 早朝] [B 朝] [C 日中]         │
│    [D 午後] [E 夕方] [F 夜]         │
│    [G 深夜]                         │
│                                     │
│    ※複数選択可                      │
├─────────────────────────────────────┤
│ ③ 曜日一括                          │  ← 既存
└─────────────────────────────────────┘
```

### 動作
- 「全時間帯OK」がデフォルトでON
- 特定グループをタップすると「全時間帯OK」がOFFになり、タップしたグループが選択される
- 複数グループを選択可能（例: A, B, Cだけ）
- 「全時間帯OK」を再タップすると全グループ選択に戻る

### データ保存
```typescript
// shift_requestsテーブルに保存
{
  staff_id: '...',
  date: '2024-02-05',
  preference: '○',
  time_slots: ['A', 'B', 'C'], // 選択した時間帯グループ、または ['ALL']
  notes: ''
}
```
